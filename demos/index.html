panel — edge-anchored positioning

Absolutely-positioned within a relatively-positioned parent; corner attributes anchor it. The combination of top / middle / bottom and left / center / right attributes determines which edge the panel snaps to. ui-window, ui-tooltip, and ui-collapsiblepanel all extend this behaviour.

Data collections

Collections are the data layer. Each holds an array of items and emits change events; UI elements bound to a collection update on those events. Below: three flavours of collection, three patterns of binding, all using the same setItemCollection hook on ui-list, ui-grid, and ui-checklist.

The simplest case — a hand-managed array. Two views (list and grid) bound to the same collection-simple stay in lockstep. Drag any row to reorder — the move propagates through the collection and the other view follows. The event log on the right shows the framework's own events firing in real time.

const items = elation.elements.create('collection-simple', {
  items: [{ name: 'Alpha' }, { name: 'Bravo' }, { name: 'Charlie' }]
});

list.setItemCollection(items);
grid.setItemCollection(items);

items.add({ name: 'Delta' });   // both views update

        

      

      
        
Swap collection-simple for collection-jsonapi and the binding doesn't change — same setItemCollection, same views, real network now. Reading .items for the first time triggers an automatic load. Reload to refetch; the list listens for collection_load_begin / collection_load and re-renders accordingly.
